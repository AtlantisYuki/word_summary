# 省流总结助手（Chrome 扩展）

一个面向网页阅读场景的省流插件，支持选区、全文、图片三种总结模式，兼容 OpenAI API 风格供应商。

## 主要功能

- 选区总结：支持快捷键、右键菜单、Popup 统一触发。
- 全文总结：页面内悬浮“省流”气泡，悬停展开摘要。
- 图片总结：右键图片可直接识图总结，结果以气泡形式展示。
- 混合总结：`更多` 页面支持文字、图片、图文混合输入。

## 交互与样式能力

- 任务前选区样式可配置：背景色、边框颜色/宽度/样式。
- 任务进行中文案可配置（默认“总结中”）。
- 结果摘要支持复制。
- 跨标签选区优先使用 Custom Highlight，减少对原始 DOM 的破坏。

## 供应商与模型配置

- 多供应商并行配置（新增/删除/启停）。
- 每个供应商支持：
  - `baseUrl`、`apiKey`
  - `textModel`（文字模型）
  - `visionModel`（视觉模型）
  - `timeoutMs`（超时）
  - `maxRetries`（失败重试次数）
  - `textPriority` / `visionPriority`（优先级）
- 支持按场景分别选择首选供应商：
  - 选区总结
  - 图片总结
  - 全文总结

## Popup / 页面结构

- Popup：
  - 三个开关：`选区总结`、`图片总结`、`全文总结`
  - 各自独立供应商选择
  - 底栏：左 `选项` / 中 `版本号` / 右 `更多`
- 选项页、更多页、关于页：统一浅色侧边导航。
- 关于页：
  - 显示当前扩展版本
  - `检查更新`（对比 GitHub Releases 最新版本，仅提示，不自动升级）
  - 项目 GitHub 地址

## 调试能力

- 可开启开发调试日志（本地存储）。
- 可查看供应商调用、重试、超时、错误详情。
- 适合排查视觉模型兼容问题与请求格式问题。

## 接口兼容说明

- 请求地址规则：`Base URL + /v1/chat/completions`
- 若 Base URL 已包含 `/v1` 或完整 `/chat/completions`，会自动兼容。
- 请求体采用 Chat Completions 风格（`model` + `messages`）。

## 本地使用

1. 打开 `chrome://extensions/`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”，选择当前项目目录
4. 进入“选项”配置供应商与模型
5. 刷新目标网页后即可使用
